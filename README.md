# 古树移栽土球吊装路径复盘系统

面向园林部门的古树移栽吊装方案三维可视化复盘工具。基于 WebGL (Three.js) 实现，无需 BIM 即可完成吊点组合评估、风载模拟、障碍碰撞检测与方案导出。

## 功能特性

### 三维交互
- **场景元素**：默认展示土球、障碍物（半透明灰蓝色）、目标树坑（绿色圆环）、候选吊点（金色吊钩）
- **吊点选择**：右侧面板点击候选吊点（最多 2 个），选中后吊钩变绿
- **吊索绘制**：选择两个吊点后自动绘制吊索与吊装路径曲线（虚线）
- **离地高度曲线**：沿吊装路径按正弦插值抬升，自动避让障碍高度
- **碰撞检测**：路径与任一障碍相交时，路径虚线、吊索、障碍物全部标红
- **摆动幅度滑块**：0–30 cm 模拟风载偏移，偏移后仍相交则提示「需调整吊点或清障」
- **吊装动画**：点击「播放吊装动画」按钮，土球沿路径实时移动，吊索动态更新

### 非三维功能
- **树体切换**：右侧面板顶部切换待移古树（内置 2 棵示例，含吨位临界场景）
- **吨位对比侧栏**：表格列出所有 6 种吊点组合的预估峰值吨位，超载或碰撞行标红
- **方案导出**：一键导出当前三维场景 PNG 截图
- **实时评估**：显示当前组合的峰值吨位、碰撞状态、风险提示

## JSON 数据字段说明

数据位于 `src/data/mockTrees.ts`，导出 `TreeData[]` 数组。单棵树结构如下：

```typescript
{
  id: "tree-001",
  name: "古银杏树（太平街）",

  soilBall: {
    diameter: 2.4,           // 土球直径（米）
    centerOfMass: { x: 0, y: 1.2, z: 0 },  // 土球重心空间坐标（米）
    weight: 18               // 土球总重（吨）
  },

  obstacles: [
    {
      id: "obs-1",
      name: "围墙",
      position: { x: 5, y: 1.5, z: 0 },   // 长方体几何中心坐标（米）
      size: { x: 0.5, y: 3, z: 8 }        // x/y/z 三方向尺寸（米）
    }
  ],

  liftPoints: [
    {
      id: "lp-1",
      name: "吊点A-东侧",
      position: { x: 8, y: 12, z: 2 },    // 吊钩坐标（米）
      ratedTonnage: 25                     // 吊点额定吨位
    }
  ],

  targetPit: {
    position: { x: 10, y: 0, z: 6 },      // 树坑中心坐标（米）
    depth: 1.8,                            // 树坑深度（米）
    diameter: 3.0                          // 树坑直径（米）
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `soilBall.diameter` | number | 土球直径（米） |
| `soilBall.centerOfMass` | Vec3 `{x,y,z}` | 土球重心空间坐标（米） |
| `soilBall.weight` | number | 土球估算重量（吨） |
| `obstacles[].position` | Vec3 | 长方体障碍物几何中心坐标 |
| `obstacles[].size` | Vec3 | 障碍物 x/y/z 三方向尺寸（米） |
| `liftPoints[].position` | Vec3 | 吊钩空间坐标 |
| `liftPoints[].ratedTonnage` | number | 吊点额定吨位 |
| `targetPit.position` | Vec3 | 目标树坑中心坐标（坑口地面） |
| `targetPit.depth` | number | 树坑深度（米） |
| `targetPit.diameter` | number | 树坑直径（米） |

## 验收场景

### 场景一：吨位不足（唯一可行方案）

**目标古树**：古槐树（公园北路）— 吨位临界（tree-002，土球 32t）

**操作步骤**：
1. 右侧「选择古树」切换到「古槐树（公园北路）— 吨位临界」
2. 在「吊点组合吨位对比」表格中查看所有 6 种组合
3. 逐一观察：由于土球重达 32t，只有吊点 G（东南，30t）参与的组合才可能满足吨位
4. 依次尝试：
   - 吊点 E (16t) + 吊点 F (16t) → 吨位不足
   - 吊点 E (16t) + 吊点 G (30t) → 吨位不足（16t 吊点超载）
   - 吊点 E (16t) + 吊点 H (14t) → 吨位不足
   - 吊点 F (16t) + 吊点 G (30t) → 吨位不足（16t 吊点超载）
   - 吊点 F (16t) + 吊点 H (14t) → 吨位不足
   - **吊点 G (30t) + 吊点 H (14t) → 吨位不足（但为最接近额定的组合）**
5. 选择吊点 G + 吊点 H，观察「当前方案评估」卡片

**预期结果**：
- 吨位对比表格中：所有组合均显示红色「风险」标签（吨位不足）
- 当前方案卡片显示预估峰值吨位 > 吊点 H 额定 14t（超载）
- 提示文字显示「吊点额定吨位不足」
- 3D 场景中吊索显示红色（吨位超载警告）
- 但路径曲线显示绿色（不与「地下管线走廊」和「门楼建筑」相交），说明这是**唯一路径可行但吨位临界**的方案

### 场景二：障碍相交

**目标古树**：古银杏树（太平街）（tree-001）

**操作步骤**：
1. 选择「古银杏树（太平街）」
2. 选择吊点 B（南侧，y=14，z=-8）+ 吊点 D（北侧，y=11，z=9）
3. 观察吊装路径曲线是否穿过中部障碍物（围墙、电力杆、景观亭）
4. 拖动「摆动幅度」滑块从 0 → 20 → 30 cm，观察状态变化
5. 切换其他吊点组合（如 A+C），对比碰撞状态

**预期结果**：
- 当路径与障碍相交时：
  - 顶部「当前方案评估」显示红色「相交」状态
  - 提示栏显示「需调整吊点或清障」
  - 3D 场景中：障碍物变深红，路径虚线变红，吊索变红
- 增大摆动幅度后（20–30 cm），原本擦边通过的组合可能从绿色安全转为红色碰撞
- 吨位对比表格中，碰撞组合标注红色「风险」标签，安全组合标注绿色「可行」

## 技术栈

- **框架**：React 18 + TypeScript + Vite 6
- **3D 渲染**：three.js（原生 WebGL，OrbitControls）
- **状态管理**：Zustand
- **样式**：Tailwind CSS 3 + lucide-react 图标
- **导出**：Three.js WebGL canvas 原生 toDataURL（PNG 截图）
- **部署**：Nginx 1.27 + Docker 多阶段构建

## 计算说明（简化模型）

- **吊装路径**：水平方向线性插值，垂直方向 `sin(π·t)·h` 正弦抬升，`h` 由两吊点中较低的吊钩高度决定（低吊点组合抬升不足易碰撞）
- **吨位估算**：按两吊索与竖直方向夹角的正弦值反比分配土球重量，乘以水平分量放大系数 1.3
- **碰撞检测**：土球等效球体，障碍等效 AABB；沿路径 40 段采样，做球-盒相交 + 线段-盒相交双重检测
- **风载偏移**：沿 (1, 0, 1) 方向施加 `swingCm / 100` 米偏移向量，叠加到土球位置后重新检测碰撞
